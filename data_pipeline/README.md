## 2.2.1 수집 데이터

LLM 기반 요구사항 생성 및 RAG(Retrieval-Augmented Generation) 구축을 위해 정책·규정, 기술 표준, 공공 가이드라인, 실제 공공사업 RFP 문서를 수집하였다.

### 수집 데이터 구성

| 구분 | 수집 대상 | 활용 목적 |
|--------|--------|--------|
| 정책/규정 | 소프트웨어 보안 약점 진단 | 보안 요구사항 및 품질 기준 확보 |
| 정책/규정 | 행정기관 및 공공기관 정보시스템 구축·운영 지침 | 공공 정보화 사업 추진 절차 및 규정 반영 |
| 정책/규정 | 정보보호 공시 | 정보보호 관련 요구사항 및 작성 기준 확보 |
| 기술 표준 | 표준 프레임워크 가이드(eGovFrame) | 공공 시스템 개발 표준 및 아키텍처 기준 확보 |
| 가이드 | 공공 SW사업 제안요청서 작성을 위한 요구사항 가이드 | 요구사항 정의 및 작성 기준 확보 |
| 가이드 | 공공데이터베이스 표준화 관리 매뉴얼 | 데이터 모델링 및 품질 관리 기준 확보 |
| 가이드 | 공공데이터 공통표준 | 데이터 표준용어 확보 |
| 가이드 | 공공정보화사업 단계별 사업관리 가이드 | 정보화사업 관리 프로세스 확보 |
| 가이드 | 소프트웨어사업 요구사항 분석 적용 가이드 | 요구사항 분석 및 품질 검증 기준 확보 |
| RFP | 세종하드웨어캠퍼스 AI 기반 공동교육플랫폼 고도화 | 실제 AI 플랫폼 구축 요구사항 학습 |


### 데이터 활용 전략

- **정책·규정 데이터**
  - 공공사업 수행 시 반드시 준수해야 하는 법적·행정적 요구사항 확보
  - 비기능 요구사항 및 제약사항 추출

- **기술 표준 데이터**
  - 시스템 설계 및 개발 표준 확보
  - 아키텍처 및 기술 요구사항 추출

- **가이드 데이터**
  - 공공 SW 사업 수행 시 요구되는 표준 프로세스 확보
  - 요구사항 품질 향상 및 분석 기준 제공




## 2.2.2 데이터 전처리 파이프라인 (RAG Knowledge Base 구축)

수집된 정책, 규정, 기술 표준, 가이드 문서는 RAG 검색 성능 향상을 위해 전처리 파이프라인을 구축하였다.

### 전처리 프로세스

```text
문서 수집
    ↓
텍스트 추출
(PDF / DOCX / TXT)
    ↓
텍스트 정제
(Cleaning)
    ↓
문서 내 섹션 분류
(장/절/항목 구조 인식)
    ↓
Chunk 분할
(1000자 기준, 150자 Overlap)
    ↓
메타데이터 생성
    ↓
Vector DB 적재
```

### 1. 문서 텍스트 추출

#### 목적
다양한 형식의 문서를 텍스트 형태로 변환

#### 수행 방법
- PDF : PyMuPDF 기반 페이지 단위 추출
- DOCX : Paragraph 기반 추출

#### 결과
- 페이지별 원문 텍스트 확보
- 문서 구조 유지

---

### 2. 텍스트 정제 (Cleaning)

#### 목적
RAG 검색 품질 향상을 위한 노이즈 제거

#### 수행 작업

- 특수문자 정리
- 중복 공백 제거
- 개행 문자 정규화
- 페이지 번호 제거
- 불필요 헤더/푸터 제거

#### 효과

- 검색 품질 향상
- 임베딩 품질 개선

---

### 3. 문서 섹션 분류

#### 목적
문서 구조 정보를 유지하여 검색 정확도 향상

#### 수행 방법

정규표현식을 활용하여:

- 제1장
- 제2절
- 1.1
- 1.1.1

등의 문서 계층 구조를 추출

#### 생성 정보

```json
{
  "chapter": "제1장",
  "section": "1.1",
  "title": "보안 요구사항"
}
```

#### 효과

- 의미 단위 기반 검색 가능
- 문맥 손실 최소화

---

### 4. Chunk 분할

#### 목적

LLM Context Window 및 RAG 검색 최적화

#### 전략

- Chunk Size : 1000자
- Overlap : 150자
- 섹션 단위 우선 분할

#### 처리 방식

1. 섹션 기준 문서 조립
2. Chunk Size 초과 시 분할
3. 문맥 단절 방지를 위한 Overlap 적용
4. 너무 짧은 Chunk는 이전 Chunk와 병합

#### 예시

```text
Chunk 1
[1~1000자]

Overlap
[851~1000자]

Chunk 2
[851~1850자]
```

---

### 5. 메타데이터 생성

#### 목적

검색 결과의 추적성 확보

#### 생성 항목

```json
{
  "source": "공공데이터 공통표준.pdf",
  "document_type": "가이드",
  "chunk_id": 125,
  "chapter": "제2장",
  "section": "2.1",
  "title": "데이터 표준"
}
```

#### 활용

- 검색 결과 출처 제공
- 요구사항 생성 시 근거 문서 제시
- Hallucination 감소
