# 자체 sLLM 개발 세부 정보

ALPLED는 공공데이터 표준 가이드라인에 부합하는 데이터베이스 설계 자동화를 지원하기 위해, 공공데이터 표준 용어 기반의 영문 약어 컬럼명 생성 sLLM을 파인튜닝하였다.

기존 범용 LLM은 비정형 한글 업무 용어를 공공데이터 표준 규칙에 맞는 영문 약어명으로 일관되게 변환하는 데 한계가 있었다. 이에 따라 **Qwen3-VL-8B-Instruct** 모델을 기반으로 **QLoRA Fine-Tuning**을 수행하여, 한글 용어를 표준 영문 약어명으로 변환하는 특화 모델을 개발하였다.

---

## 1. 모델 목적

| 구분    | 내용                                              |
| ----- | ----------------------------------------------- |
| 모델 목적 | 비정형 한글 업무 용어를 공공데이터 표준 용어 규칙에 맞는 영문 약어명으로 자동 변환 |
| 적용 대상 | ERD 설계서 Agent, 데이터베이스 설계서 Agent                 |
| 주요 활용 | 테이블명/컬럼명 표준화, DB 컬럼명 자동 생성, ERD 및 DB 설계 자동화 지원  |
| 입력    | 한글 업무 용어 또는 컬럼 의미                               |
| 출력    | 공공데이터 표준 기반 영문 약어명, Snake Case 형태               |
| 예시    | 공표일자 → OFANC_YMD                                |

---

## 2. 주요 기능

* 비정형 한글 용어 자동 분절
* 공공데이터 공통표준용어 기반 의미 매핑
* 표준 영문 약어명 생성
* Snake Case 기반 DB 컬럼명 생성
* ERD 설계서 및 데이터베이스 설계서의 컬럼명 표준화 지원
* 학습 데이터에 포함되지 않은 복합 용어에 대한 일반화 변환 지원

## 3. 모델 개발 구성도

```text
공공데이터 공통표준용어 데이터
        ↓
데이터 전처리
- 한글 용어 / 영문 약어 쌍 구성
- ChatML 메시지 형식 변환
- '_' 토큰 분리 전처리
        ↓
데이터셋 분할
- Train 10,000건
- Validation 2,000건
- Test 1,000건
        ↓
기반 모델 로드
Qwen3-VL-8B-Instruct
        ↓
4bit Quantization
BitsAndBytes NF4
        ↓
QLoRA Adapter 구성
- LoRA rank: 16
- LoRA alpha: 32
- target modules:
  q_proj, k_proj, v_proj, o_proj,
  gate_proj, up_proj, down_proj
        ↓
Fine-Tuning
- epoch: 3
- batch size: 4
- gradient accumulation: 4
- learning rate: 2e-5
        ↓
평가
Accuracy / Precision / Recall / BLEU
        ↓
ALPLED Agent 연동
ERD · DB 설계 자동화에서 표준 컬럼명 생성에 활용
```

## 6. 학습 성능 평가

| Metric     | Score  |
| ---------- | ------ |
| Accuracy   | 77.40% |
| Precision  | 63.14% |
| Recall     | 63.18% |
| BLEU Score | 90.06% |

파인튜닝 결과, 공공데이터 표준 용어 규칙을 기반으로 한 영문 약어명 생성 성능이 향상되었다. 특히 BLEU Score 90.06%를 기록하여 정답 약어명과 높은 문자열 유사도를 보였으며, 학습 데이터에 포함되지 않은 복합 한글 용어에 대해서도 일정 수준의 일반화 성능을 확인하였다.

---

## 7. ALPLED 내 활용 방식

파인튜닝된 sLLM은 ALPLED의 멀티 Agent 산출물 생성 흐름 중 **ERD 설계서**와 **데이터베이스 설계서** 생성 단계에서 활용된다.

1. 요구사항 정의서와 회의록에서 데이터 항목 후보 추출
2. RAG를 통해 공공데이터 표준 용어 및 도메인 후보 검색
3. 파인튜닝 sLLM을 통해 한글 데이터 항목을 표준 영문 약어명으로 변환
4. 최종 ERD JSON 및 데이터베이스 설계서 DOCX에 표준 컬럼명 반영

이를 통해 산출물 생성 과정에서 **DB 컬럼명 표준화**, **명명 규칙 일관성 확보**, **공공데이터 표준 준수 수준 향상**을 지원한다.
