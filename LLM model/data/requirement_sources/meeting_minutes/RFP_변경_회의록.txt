변경 요구사항 검토 회의록

1. 회의 개요
- 일시: 2026년 7월 15일 오전 10시
- 참석자: 최발주(발주처 담당), 이기획(PM), 김개발(개발팀장)

2. 발주처 요구사항 변경 요청
가. 성능: 데이터 처리 속도를 기존 1초 이내에서 0.5초 이내로 단축 요청
나. 기능: AI 기반 상담 서비스에 음성 인식 기능 추가 요청
다. 정책: 개인정보 보호 강화 차원에서 사용자 데이터 암호화 범위 확대 요청
라. 화면: 사용자 편의성을 위해 모바일 최적화 UI 추가 요청
마. 데이터: 통합 플랫폼 내 기존 금융 데이터 외에 외부 신용정보 데이터 연동 요청

3. 개발팀장 검토 의견 (김개발)
- 성능 변경: 데이터 처리 속도 개선을 위해 서버 증설 및 알고리즘 최적화 필요, 약 2개월 일정 추가 예상
- 기능 추가: 음성 인식 기능은 신규 모듈 개발과 AI 학습 데이터 수집이 필요하여 3개월 추가 소요 예상
- 정책 변경: 암호화 범위 확대는 보안 모듈 수정 및 테스트 강화 필요, 약 1개월 일정 영향
- 화면 변경: 모바일 UI 개발은 기존 웹 UI와 별도로 개발해야 하므로 1.5개월 추가
- 데이터 연동: 외부 신용정보 연동은 API 연동 및 데이터 정합성 검증 필요, 2개월 일정 소요

4. PM 최종 합의안 (이기획)
- 성능과 기능 변경은 프로젝트 핵심 요소이므로 반영하되 일정 조정 필요
- 정책 및 화면 변경은 보안과 사용자 편의성 제고를 위해 반드시 반영
- 데이터 연동은 추가 검토 후 2단계 개발로 진행하는 방안 제안
- 전체 일정 약 4개월 연장 조정 및 추가 예산 확보 필요

5. 합의된 변경사항 정리
- 변경 항목:
  1) 성능: 데이터 처리 속도 1초 이내 -> 0.5초 이내 단축
  2) 기능: AI 상담 서비스에 음성 인식 기능 추가
  3) 정책: 사용자 데이터 암호화 범위 확대
  4) 화면: 모바일 최적화 UI 추가

회의를 마치며, 변경 사항에 따른 상세 일정 및 예산 계획은 별도 문서로 작성하여 공유하기로 함.
