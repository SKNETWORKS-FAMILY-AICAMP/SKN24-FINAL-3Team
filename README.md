## 📌 프로젝트 소개

본 프로젝트는 **AI-DLC(AI 협업 소프트웨어 개발 방법론)**를 기반으로 소프트웨어 개발 생명주기에서 반복적으로 작성되는 주요 개발 산출물을 자동 생성하는 플랫폼입니다.

기존 SDLC 방식에서는 요구사항 분석, 화면 설계, 아키텍처 설계, 데이터 모델링, 테스트 시나리오 작성 등 다양한 문서를 수작업으로 작성해야 합니다. 이 과정에서 문서 작성 시간이 과도하게 소요되고, 요구사항 변경 시 여러 산출물을 다시 수정해야 하는 문제가 발생합니다.

본 플랫폼은 이러한 문제를 해결하기 위해 **RFP, 회의록, 프로토타입 이미지, 아키텍처 설정** 등을 입력받고, 산출물별 전문 Agent가 협업하여 각 산출물을 자동 생성합니다.

<br />

## ❗ 문제 정의

기존 프로젝트 환경에서는 사용자 요구사항 정의서, 사용자 인터페이스 설계서, 통합시험 시나리오 등이 각각 개별적으로 작성됩니다. 이로 인해 다음과 같은 문제가 발생합니다.

| 구분 | 문제 내용 | 영향 |
|---|---|---|
| 문서 작성 시간 과다 | 단계별 산출물을 수작업으로 작성 | 일정 지연 및 업무 부담 증가 |
| 요구사항 누락 | RFP나 회의록의 요구사항이 일부 문서에 반영되지 않음 | 기능 누락 및 재작업 발생 |
| 산출물 간 불일치 | 요구사항, UI, DB, 테스트 문서 간 표현과 필드가 다름 | 검토 비용 증가 |
| 변경 관리 어려움 | 기획 변경 시 관련 문서 수정 범위 파악이 어려움 | 문서 최신성 저하 |
| 품질 편차 | 작성자의 경험과 이해도에 따라 문서 품질 차이 발생 | 산출물 신뢰도 저하 |
| 보안 위험 | 외부 LLM 사용 시 기업 내부 데이터 유출 가능성 존재 | 폐쇄망·보안 환경 적용 어려움 |

## 💡 해결 방안

본 프로젝트는 산출물별 전문 Agent를 구성하여 입력된 자료를 분석하고, 개발 단계별 산출물을 자동 생성합니다.

```text
RFP / 회의록 / 프로토타입 이미지 / 아키텍처 설정
        ↓
입력 데이터 전처리 및 구조화
        ↓
사용자 요구사항 정의서 Agent 실행
        ↓
사용자 요구사항 정의서 생성
        ↓
산출물별 전문 Agent 실행
        ↓
사용자 인터페이스 설계서 / 아키텍처 설계서 / 엔티티 관계 모형 설계서 / 데이터베이스 설계서 / 통합시험 시나리오 생성
        ↓
산출물 간 정합성 검증
        ↓
최종 개발 산출물 제공
```

## 🧩 주요 기능

### 1. 요구사항 자동 분석

- RFP와 회의록에서 핵심 요구사항 추출
- 기능 요구사항과 비기능 요구사항 분류
- 요구사항 ID 부여
- 검증 기준 자동 정리

### 2. 개별 산출물 자동 생성

- 사용자 요구사항 정의서
- 사용자 인터페이스 설계서
- 아키텍처 설계서
- 엔티티 관계 모형 설계서
- 데이터베이스 설계서
- 통합시험 시나리오

### 3. 멀티 에이전트 기반 문서 생성

- 산출물별 전문 Agent 구성
- 선행 산출물을 기반으로 후속 산출물 생성
- Agent 간 결과 공유 및 문서 흐름 연결

### 4. 산출물 정합성 검증

- 요구사항 ID 기반 추적성 확보
- 요구사항, 화면, DB, 테스트 항목 간 누락 여부 확인
- 문서 간 용어 및 필드 일관성 검토

### 5. 자체 sLLM 기반 보안 환경 지원

- 기업 내부 데이터 보호
- 폐쇄망 환경 적용 가능성 확보
- 외부 LLM 의존도 감소

## 🤖 Agent 구성

| Agent | 입력 | 출력 | 역할 |
|---|---|---|---|
| 사용자 요구사항 정의서 Agent | RFP, 회의록 | 사용자 요구사항 정의서 | 요구사항 추출, 분류, ID 부여, 검증 기준 생성 |
| 사용자 인터페이스 설계서 Agent | 요구사항 정의서,<br/> 프로토타입 이미지 | 사용자 인터페이스 설계서 | 화면 구조, UI 구성요소, 이벤트, 요구사항 매핑 생성 |
| 아키텍처 설계서 Agent | 요구사항 정의서,<br/> 아키텍처 설정 정보 | 아키텍처 설계서 | 시스템 구성, 인프라, 보안, 연계 구조 문서화 |
| 엔티티 관계 모형 설계서 Agent | 요구사항 정의서 | 엔티티 관계 모형 설계서 | 엔티티, 속성, 식별자, 관계 도출 |
| 데이터베이스 설계서 Agent | 요구사항 정의서,<br/> 엔티티 관계 모형 설계서 | 데이터베이스 설계서 | 테이블, 컬럼, 키, 제약조건, 데이터 타입 정의 |
| 통합시험 시나리오 Agent | 요구사항 정의서,<br/> 사용자 인터페이스 설계서 | 통합시험 시나리오 | 테스트 항목, 입력값, 절차, 예상 결과 생성 |

## 🌟 프로젝트 차별점

### 1. 문서별 전문 Agent 구조

각 문서 유형에 맞는 전문 Agent를 구성하여 산출물의 목적과 형식에 맞는 결과를 생성합니다.

### 2. 요구사항 중심 자동화

모든 산출물은 사용자 요구사항 정의서를 기준으로 생성되며, 요구사항 ID를 통해 추적성을 확보합니다.

### 3. 기업 보안 환경 고려

외부 LLM 사용이 어려운 기업 환경을 고려하여 자체 sLLM 기반 구조를 지향합니다.

### 4. 정합성 검증 확장 가능성

향후 요구사항 ID, 화면 ID, 테이블 ID, 테스트 케이스 ID를 연결하여 산출물 간 정합성을 검증할 수 있습니다.
